# Phase 04 Execution Report

Date: 2026-06-27

## Summary

Phase 04 implemented reproducible modeling evidence for checklist IDs 10-13. The work adds multi-seed/fold SASHTS benchmark metrics, stronger non-GNN baselines, ablation outputs, full PyTorch GraphSAGE metrics, and a multi-dataset trace experiment over SASHTS plus two SocioPatterns contact datasets.

The external multi-dataset evidence uses SocioPatterns primary school and hospital contact graphs. Both are preprocessed as weighted contact graphs and evaluated with source-tracing tasks generated by SI simulation, where the true source is known by construction.

`results/metrics/trace_summary_table.csv`

- Multi-dataset source-tracing summary over SASHTS, SocioPatterns primary school, and SocioPatterns hospital.
- Trace tasks are generated by SI simulation on real weighted contact graphs.
- True source is known by construction.
- Methods include random, degree, weighted degree, closeness, betweenness, center, and leave-one-dataset-out GraphSAGE-style mean-neighbor logistic ranker.

`results/model/full_pytorch_graphsage_summary.csv`

- Full PyTorch two-layer weighted mean GraphSAGE.
- 15 runs: 5 seeds x 3 household-grouped folds.
- Validation AP early stopping with checkpointed best state per seed/fold.
- Selected validation-CV configuration: hidden dimension 16, dropout 0.20/0.10, learning rate 0.01, weight decay 0.0001.
- Test AUC `0.5772 +/- 0.0443`.
- Test AP `0.7679 +/- 0.0246`.
- Test F1 `0.8265 +/- 0.0121`.

`results/model/gap_summary.csv`

- Train/validation/test gap summary for full PyTorch GraphSAGE across 15 runs.
- Train-test AP gap `0.1186 +/- 0.0777`.
- Validation-test AP gap `0.0392 +/- 0.0412`.
- Train-test AUC gap `0.1892 +/- 0.1518`.
- Validation-test AUC gap `0.0483 +/- 0.0610`.

`scripts/28_phase04_torch_hyperparameter_tuning.py`

## Design Choices

- Used household-grouped folds for SASHTS to avoid household leakage.
- Used five seeds and three folds rather than one cherry-picked seed.
- Used validation average precision for early stopping and checkpoint selection in trainable models.
- The initial `numpy`/`pandas` mean-neighbor model remains as a lightweight reproducible comparator.
- The full PyTorch two-layer GraphSAGE has now been run through a user CMD environment with `torch` installed.
- Hyperparameter tuning selected the final GraphSAGE configuration by validation AP before the final full rerun.
